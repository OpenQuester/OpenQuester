# Decision navigation

Use this file before broad project decisions.

## Rule

Do not start with code for broad decisions. First collect context, read the right docs, and produce a plan.

## What to read

- Product work: `docs/product/00-north-star.md`, `docs/product/01-release-plan.md`.
- Gameplay work: `docs/specs/game-state-matrix.md`, relevant `server/docs/*`, relevant backend code.
- Frontend work: `client/AGENTS.md`, `.agents/skills/frontend-game-ui-state/SKILL.md`, Context7 docs for Flutter/Dart and touched packages.
- Backend work: `server/AGENTS.md`, `docs/architecture/adr/`, relevant backend skill.
- Package/editor work: `.agents/skills/package-editor-change/SKILL.md`, package specs.
- Validation work: `.agents/skills/project-assurance/SKILL.md`, `docs/agent/03-verification-matrix.md`.
- Docs work: `.agents/skills/docs-upkeep/SKILL.md`, `docs/agent/02-source-of-truth.md`.

## Plan shape

For broad work, provide goal, non-goals, current state with file evidence, constraints, proposed structure, risks, verification plan, and docs that need updates.
